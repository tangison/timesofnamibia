# Contributing to Times of Namibia (TANGISON Publication)

Thanks for your interest in contributing. This document covers the basics.

## Development setup

See the [root README](./README.md) for installation and run instructions.

## Branching & PRs

1. Branch from `main`:
   - `feat/<short-description>` for new features
   - `fix/<short-description>` for bug fixes
   - `chore/<short-description>` for tooling, deps, refactors
   - `docs/<short-description>` for documentation
2. Keep branches short-lived (< 1 week). Rebase onto `main` before opening a PR.
3. Open a PR with a clear description. Link any issues it closes.
4. CI must pass before merge.
5. Squash-merge. Use [Conventional Commits](https://www.conventionalcommits.org/):
   - `feat: add job email alerts`
   - `fix: prevent auth bypass on /api/proxy`
   - `chore: bump Next.js to 16.2`
   - `docs: rewrite backend README`

## Code style

### Frontend (TypeScript / React / Next.js)

- `strict: true` in `tsconfig.json` — no implicit `any`, no untyped function components.
- Prefer **server components** by default. Add `"use client"` only when you need state, effects, or browser APIs.
- Use `next/link` for internal navigation — never raw `<a>` to `/` routes (ESLint will warn).
- Validate every POST/PUT body with a Zod schema. See `src/app/api/newsletter/route.ts` for the pattern.
- For admin-only endpoints, gate with `requireAdmin(request)` from `@/lib/auth`. See `src/app/api/data-agent/route.ts`.
- Add rate limiting to any endpoint that mutates state or triggers work. Use `rateLimit(request, { windowMs, max })` from `@/lib/auth`.
- Prefer `next/image` over raw `<img>`. If you must use `<img>`, always set `width`, `height`, `loading`, and `alt`.
- **Domain types live in `src/lib/types.ts`.** Don't define Article/Job/Tender interfaces elsewhere — import from there. Legacy types in `ton-data.ts` are deprecated.

### Backend (Python / FastAPI)

- Type-annotate every function signature.
- Use Pydantic v2 models for request and response bodies. Declare `response_model=` on every route.
- Validate at the boundary (Pydantic schema) — keep handlers thin.
- Use `app.services.repository.SourceItemRepository` for DB access — do not reach into `Session` directly from route handlers.
- **Use `hmac.compare_digest()` for secret comparisons** (see `backend/app/api/ingest.py:_verify_secret`). Never use `!=` for secrets — it's vulnerable to timing attacks.
- Run `ruff check app/` and `ruff format app/` before pushing.

### Convex (TypeScript backend)

- **Admin mutations** must call `requireAdmin(args.adminToken)` at the top of the handler. See `convex/mutationsAdmin.ts` for the pattern.
- **Public mutations** must NEVER accept `userId`, `verified`, `role`, or any other trust-bearing field from the client. Use `ctx.auth.getUserIdentity()` for identity once Convex Auth is wired up.
- **Public queries** that return user-generated content must be admin-gated (return `[]` or `null` without a valid `adminToken`). See `convex/queries.ts:listWireSubmissions` and `getSubscriberByEmail`.
- The `convex/_generated/` directory is auto-generated by `npx convex dev` — **never edit by hand**. The current stubs are placeholders that will be overwritten.

## Testing

- **Frontend**: add tests alongside the code in `src/**/__tests__/*.test.ts`. Run `npm test` before pushing. Coverage target: 60% for `src/lib/` and `src/app/api/`.
- **Backend**: add tests in `backend/tests/test_*.py`. Use the `client` fixture from `backend/tests/conftest.py`. Run `pytest` before pushing.
- **Convex**: TODO — add tests via `convex-test` once Convex is provisioned.

## Security

- Never commit `.env`, `*.db`, `*.db-wal`, `*.db-shm`, or any file containing real secrets.
- Never log secrets, tokens, or full request bodies that may contain PII.
- Never disable the auth check on an endpoint that mutates state.
- **JSON-LD**: always use the `safeJsonLd()` helper (escapes `<` to `\u003c`) when serializing JSON-LD via `dangerouslySetInnerHTML`. See `src/app/layout.tsx` for the helper.
- If you discover a security issue, do NOT open a public PR — email security@timesofnamibia.com (or open a private vulnerability advisory on GitHub).

## Performance

- Never use `cache: "no-store"` without a specific reason — it disables all HTTP caching. Default to `revalidate = 300` for content pages.
- Never fetch all rows from a table to filter in application code — push the filter into SQL (or Convex `.withIndex()`).
- Never run an LLM call before checking for duplicates.
- Add `Cache-Control` headers to GET API routes where the response is cacheable (e.g. `s-maxage=60, stale-while-revalidate=300`).

## SEO

- **Internal pages must be `noindex`**: any page under `/brand`, `/plan`, `/business-plan`, `/documents`, `/tangison`, `/contributor`, `/maintenance`, plus `not-found.tsx`, must have `robots: { index: false, follow: true }` in their metadata export.
- **Article pages use ISR**: `export const revalidate = 300;` — never `force-dynamic`.
- **Title template**: don't manually append " — Times of Namibia" to titles. The layout template does that automatically.
- **JSON-LD**: use `safeJsonLd()` for all `dangerouslySetInnerHTML` JSON-LD blocks.
- **Breadcrumb URLs**: use `/section/SECTION` not `/SECTION` (the latter 404s).

## Pull request checklist

- [ ] Branch is up to date with `main`
- [ ] `npm run lint` passes (0 errors)
- [ ] `npm run typecheck` passes
- [ ] `npm test` passes
- [ ] `npm run build` succeeds
- [ ] Backend: `ruff check app/` passes
- [ ] Backend: `pytest` passes
- [ ] No new `any` types introduced without justification
- [ ] No new `'unsafe-eval'` in CSP
- [ ] No new `cache: "no-store"` without justification
- [ ] New API routes have Zod validation (frontend) or Pydantic (backend)
- [ ] New admin endpoints are gated with `requireAdmin()` or `requireAdmin(args.adminToken)`
- [ ] New Convex mutations don't accept `userId`/`verified`/`role` from the client
- [ ] New pages that are internal-only have `robots: { index: false }`
- [ ] New `dangerouslySetInnerHTML` JSON-LD uses `safeJsonLd()`
- [ ] No secrets, `.env`, or DB files committed
